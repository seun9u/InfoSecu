# 📘 C언어 기초 핵심 요약

## 1. 프로그램의 흐름과 `main()` 함수
C언어 프로그램은 **`main()` 함수로 시작하여 `main()` 함수가 종료되면 프로그램도 함께 종료**됩니다. 즉, `main()`은 프로그램의 진입점(Entry Point)입니다.

### C언어 기본 코드 구조
```c
#include <stdio.h> // 표준 입출력 헤더 파일 포함

int main() {
    // 코드가 실행되는 영역
    return 0; // 프로그램의 정상 종료를 의미
}
```

---

## 2. 프로젝트와 `main()` 함수의 규칙
> **⚠️ 중요:** 하나의 프로젝트(실행 파일)에는 **반드시 단 하나의 `main()` 함수만 존재**해야 합니다.

* 연습 시 여러 소스 파일(.c)에 각각 `main()` 함수를 만들면 **컴파일(링크) 에러**가 발생합니다.
* 5개의 소스 파일이 있더라도 실행되는 `main()`은 오직 하나여야 합니다.

### 💡 여러 파일에서 연습할 때 해결 방법
1. **주석 처리:** 실행할 파일의 `main()`만 남기고, 나머지 파일의 코드는 모두 주석 처리합니다.
2. **빌드 제외 (Visual Studio 기준):**
    * 제외할 파일 우클릭 -> **속성 (Alt + F7)** -> **일반** -> **[빌드에서 제외]** 항목을 `예`로 설정.

## 3. 주석 (Comments)
코드에 대한 설명을 적거나, 잠시 실행을 막을 때 사용합니다.

* `//`: **한 줄 주석** (해당 줄의 끝까지 무시)
* `/* ... */`: **여러 줄 주석** (블록 단위로 무시)

---

## 4. 출력 시스템 (Output)
`printf()` 함수는 데이터를 **표준 출력(stdout)** 스트림으로 보냅니다.

| 종류 | 설명 | 대상 |
| :--- | :--- | :--- |
| **stdout** | 표준 출력 (Standard Output) | 정상적인 프로그램 결과가 출력되는 화면 |
| **stderr** | 표준 에러 (Standard Error) | 오류 메시지가 출력되는 화면 (별도로 리다이렉션 가능) |

---

## 5. 메모리 구조 (Memory Layout)
프로그램이 실행될 때 운영체제로부터 할당받는 메모리 공간은 다음과 같이 구분됩니다.

* **코드 (Code/Text):** 실행할 프로그램의 코드가 저장되는 영역.
* **데이터 (Data):** 전역 변수, 정적(static) 변수 중 **초기화된** 변수.
* **BSS:** 전역 변수, 정적 변수 중 **초기화되지 않은** 변수 (0으로 자동 초기화).
* **힙 (Heap):** 사용자가 동적으로 할당(`malloc` 등)하고 해제하는 영역.
* **스택 (Stack):** **지역 변수**와 매개 변수가 저장되는 영역.
    * `main()` 함수나 다른 함수 내에서 선언된 변수들이 이곳에 해당하며, 함수 호출이 끝나면 사라집니다.

## 6. 터미널 및 컴파일 명령어 (Linux/Unix 환경)

### 프롬프트 설정 및 화면 제어
* **프롬프트 변경:** `PS1=" \n$ "` (줄바꿈 후 $ 표시로 변경)
* **화면 지우기:** `$ clear`

### GCC 컴파일 방법
`gcc`를 사용하여 소스 코드를 실행 파일로 변환합니다.

```bash
# 기본 형식
gcc -o [실행파일명] [소스파일명]

# 예시 (hello.c를 컴파일하여 hello라는 실행 파일 생성)
gcc -o hello hello.c
```
