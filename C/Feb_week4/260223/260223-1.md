# 📑 일반 배열 vs 포인터 배열 (Pointer Array)

## 1. 기본 개념 비교

### 1) 일반 배열 (Array)
* **정의:** 동일한 자료형의 **데이터(값)**를 연속적으로 저장하는 공간입니다.
* **특징:** 마지막에 문자열의 끝을 알리는 **`\0` (NUL 문자, 1byte)**가 존재합니다.
* **예시:** `int buf[5];` (정수 5개 저장, 20byte)

### 2) 포인터 배열 (Pointer Array)
* **정의:** **주소(Pointer)**를 저장하는 배열입니다. 즉, 배열의 각 요소가 포인터 변수입니다.
* **형식:** `자료형 *변수명[개수];`
* **특징:** 포인터 배열의 마지막 요소로는 주로 **`NULL` (널 포인터)**를 사용하여 배열의 끝을 표시합니다.
* **크기:** 시스템의 주소 버스 크기에 따라 결정됩니다.
    * **32bit 시스템:** 주소값이 4byte이므로 `sizeof`는 `4 * 개수`
    * **64bit 시스템:** 주소값이 8byte이므로 `sizeof`는 `8 * 개수`

---

## 2. 시스템 아키텍처별 크기 확인 (`array13.c`)

64bit 환경에서 `-m32` 옵션을 사용하여 32bit 바이너리로 컴파일했을 때의 크기 차이를 확인합니다.

```c
// array13.c
#include <stdio.h>

int main(){
    char *cbuf[5];  // 주소 5개를 저장하는 배열
    int  *ibuf[5];

#ifdef D64bit
    printf("64bit - cbuf size: %ld \n", sizeof(cbuf)); // 8 * 5 = 40
    printf("64bit - ibuf size: %ld \n", sizeof(ibuf)); // 8 * 5 = 40
#endif

#ifdef D32bit
    printf("32bit - cbuf size: %d \n", sizeof(cbuf));  // 4 * 5 = 20
    printf("32bit - ibuf size: %d \n", sizeof(ibuf));  // 4 * 5 = 20
#endif
    return 0;
}
```

**컴파일 결과:**
* **64bit:** `$ gcc -Wall -DD64bit -o array13 array13.c` -> **40byte** 출력
* **32bit:** `$ gcc -Wall -DD32bit -m32 -o array13 array13.c` -> **20byte** 출력

## 3. 포인터 배열의 초기화와 메모리 구조 (`array14.c`)

포인터 배열은 실제 문자열 데이터가 아닌, 문자열이 저장된 **메모리의 주소**를 들고 있습니다.



### 소스 코드
```c
// array14.c
#include <stdio.h>

int main(){
    char array[] = "Hello C!"; // 일반 문자 배열
    char *parray[] = { "Linux", "Unix", "Windows", "Mac", NULL }; // 포인터 배열

    // 포인터 배열 출력
    for(int i = 0; parray[i]; i++) // parray[i]가 NULL(0)이 아닐 때까지 반복
        printf("parray[%d] = %s\n", i, parray[i]);

    return 0;
}
```

### 📍 메모리 구조 도식 (64bit 기준)
`parray`라는 아파트(배열)의 각 호수에는 실제 사람이 사는 게 아니라, 사람이 사는 다른 집의 **'주소록'**이 들어있는 것과 같습니다.

```text
       parray (Stack 영역)           실제 문자열 데이터 (ROData 영역)
       [0]   [1]   [2]   [3]   [4]
       +-----+-----+-----+-----+--------------------+
       | 100 | 200 | 300 | 400 | 0x0000000000000000 | (NULL)
       +-----+-----+-----+-----+--------------------+
          |     |     |     |
          |     |     |     +--> 0x...6017 "Mac\0"
          |     |     +--------> 0x...600f "Windows\0"
          |     +--------------> 0x...600a "Unix\0"
          +--------------------> 0x...6004 "Linux\0"
```

## 🐞 4. GDB 정밀 분석 로그

GDB를 통해 포인터 배열 내부의 주소값과 `NULL` 포인터를 확인합니다.

### 1) 포인터 배열의 요소별 주소값 확인 (`x/1xg`)
```gdb
(gdb) p parray
$7 = {0x555555556004 "Linux", 0x55555555600a "Unix", 0x55555555600f "Windows", 0x555555556017 "Mac", 0x0}

# &parray[0] 부터 8바이트(g, giant word)씩 주소값을 검사
(gdb) x/1xg &parray[0]
0x7fffffffe190: 0x0000555555556004  # Linux의 시작 주소
(gdb) x/1xg &parray[4]
0x7fffffffe1b0: 0x0000000000000000  # NULL 포인터 (배열의 끝)
```

### 2) 주소를 찾아가서 문자열 확인 (`x/s`)
```gdb
(gdb) x/s parray[0]
0x555555556004: "Linux"

# NULL 포인터(0x0) 접근 시 에러 발생 확인
(gdb) x/s parray[4]
0x0:    <error: Cannot access memory at address 0x0>
```

---

## 💡 핵심 요약
1.  **배열**은 값을 담고, **포인터 배열**은 주소값들을 담는다.
2.  포인터 배열의 크기는 CPU 아키텍처(32/64bit)에 따라 포인터 한 칸의 크기가 결정되므로 달라진다.
3.  `for(int i=0; parray[i]; i++)` 처럼 조건식에 `parray[i]`를 넣으면, 마지막 요소인 **NULL(0)**을 만났을 때 **거짓(False)**이 되어 반복문이 종료된다.
