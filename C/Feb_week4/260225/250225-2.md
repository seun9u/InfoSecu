## 실습.[LAB] 분할 컴파일하기

### 1. 분할 컴파일이란
* 소스(.c)를 여러 개로 나누어서 파일을 각각 컴파일한 후 오브젝트 파일( ~~.o )을 생성한다.
* 생성된 오브젝트 파일들을 합쳐서 링크(Link)하여 하나의 실행파일로 만드는 방식이다.
* 분할 컴파일은 함수를 사용해서 작성해야 한다.

### 2. 분할 컴파일의 장점
* **역할 분리(모듈화):** 기능별로 `add_addr.c`, `search_addr.c`, `del_addr.c` 처럼 분리해서 작성할 수 있다.
* **재사용성 증가:** 작성된 모듈을 다른 프로젝트에서도 그대로 가져다 쓸 수 있으며, 라이브러리(.a/.so, .lib/.dll)로 만들어 사용한다.
* **유지보수/확장 용이:** 버그 발생 시 해당 모듈만 집중 분석하여 수정할 수 있고, 기능 추가 시 영향 범위를 최소화할 수 있다.
* **빌드 시간 단축 (핵심 장점):** 코드가 수정된 .c 파일만 다시 컴파일하고 나머지 .o는 재사용 가능하다. 프로젝트가 커질수록 "전체 재컴파일" 시간이 줄어든다.
* **협업에 유리:** 파일 단위로 작업 분담이 가능하여 여러 프로그래머가 동시에 작업하기 좋다.

**※ 주의:** 분할 컴파일을 할 때는 프로젝트 1개당 반드시 **main() 함수는 1개만 존재**해야 한다.

---

### 3. 주소록 프로그램의 분할 컴파일 파일 예시
| 파일명 | 포함 함수 | 역할 |
| :--- | :--- | :--- |
| **main.c** | `main()` | 프로그램 시작 및 흐름 제어 |
| **add_addr.c** | `add_addr()` | 주소록 추가 |
| **search_addr.c** | `search_addr()` | 주소록 검색 |
| **del_addr.c** | `del_addr()` | 주소록 삭제 |
| **view_addr.c** | `view_addr()` | 주소록 확인 |
| **menu_addr.c** | `menu_addr()` | 메뉴 표시 |

---

### 4. 실습 코드 구현


#### [파일 1] main.c
```c
#include <stdio.h>

void test_func1();
void test_func2();

int main()
{
    test_func1();
    test_func2();

    return 0;
}
```

#### [파일 2] test_func1.c
```c
#include <stdio.h>

void test_func1()
{
    printf("test_func1() 호출!\n");
}
```

#### [파일 3] test_func2.c
```c
#include <stdio.h>

void test_func2()
{
    printf("test_func2() 호출!\n");
}
```

---

### 5. 컴파일 및 실행 과정


#### Step 1: 오브젝트 파일 생성
```bash
$ gcc -c main.c test_func1.c test_func2.c
```

#### Step 2: 링크 및 실행 파일 생성
```bash
$ gcc -o main main.o test_func1.o test_func2.o
```

#### Step 3: 실행 결과 확인
```bash
$ ./main
test_func1() 호출!
test_func2() 호출!
```
