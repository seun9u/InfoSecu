# 🔍 GDB 명령행 인자(argv) 메모리 정밀 분석

GDB 내에서 `set args` 명령어를 사용하여 프로그램 실행 시 인자를 전달하고, `argv` 포인터 배열이 메모리 상에서 어떻게 실제 문자열 주소를 저장하고 있는지 분석합니다.

---

## 1. GDB 실행 및 인자 설정 (set args)

GDB를 실행한 후, 프로그램에 전달할 인자를 미리 설정할 수 있습니다.

```gdb
$ gdb -q ./arguments
(gdb) set args 1 2 3   # 인자로 1, 2, 3을 설정 (argc는 4가 됨)
(gdb) b main           # 중단점 설정
(gdb) r                # 프로그램 실행
(gdb) disp i           # 변수 i 자동 출력
(gdb) disp argv[i]     # 포인터 배열 argv의 현재 요소 자동 출력
```

---

## 2. GDB 분석 로그 및 단계별 흐름

```gdb
(gdb) n
9               printf("argc = %d\n", argc);
1: i = 0
2: argv[i] = 0x7fffffffe56b "/home/linuxadmin/.../arguments"

(gdb) n
argc = 4
12          for(; i < argc; i++)
1: i = 0
2: argv[i] = 0x7fffffffe56b "/home/linuxadmin/.../arguments"

# --- 루프 반복을 통해 argv[0] ~ argv[3]까지 출력되는 과정 ---
# i = 1 일 때 -> argv[1] = 0x...e59f "1"
# i = 2 일 때 -> argv[2] = 0x...e5a1 "2"
# i = 3 일 때 -> argv[3] = 0x...e5a3 "3"

(gdb) n
15          return 0;
1: i = 4
2: argv[i] = 0x0  # argv[4]는 NULL 포인터임을 확인
```

---

## 3. argv 메모리 구조 도식화

`argv`는 주소값을 담는 8바이트(64bit 기준) 크기의 방들이 연속된 배열이며, 각 방에 저장된 주소는 실제 문자열 데이터가 저장된 위치를 가리킵니다.



```text
       argv 포인터 배열 (Stack 영역)
       0x...e2c8      0x...e2d0       0x...e2d8      0x...e2e0      0x...e2e8
       [0]            [1]             [2]            [3]            [4]
       +--------------+---------------+--------------+--------------+------------------+
       | 0x...e56b    | 0x...e59f     | 0x...e5a1    | 0x...e5a3    | 0x0000...0000    |
       +--------------+---------------+--------------+--------------+------------------+
       |              |               |              |  
       |              |               |              +-------------------+
       |              |               +-----------------------+          |
       |              +------------------------+              |          |
       +-----------------------+               |              |          |
                               v               v              v          v
                       +-------+---+   +---+---+      +---+---+      +---+---+
                       |/ho..\0|       | 1 |\0 |      | 2 |\0 |      | 3 |\0 |
                       +-------+---+   +---+---+      +---+---+      +---+---+
                       0x...e56b       0x...e59f      0x...e5a1      0x...e5a3
```

## 4. GDB 메모리 검사 (x 명령어)

`x/6xg argv` 명령어를 통해 `argv` 주소부터 8바이트(`g`)씩 16진수(`x`)로 6개의 데이터를 확인합니다.

### 1) 포인터 배열 자체의 내용 확인
```gdb
(gdb) x/6xg argv
0x7fffffffe2c8: 0x00007fffffffe56b      0x00007fffffffe59f  # argv[0], argv[1]
0x7fffffffe2d8: 0x00007fffffffe5a1      0x00007fffffffe5a3  # argv[2], argv[3]
0x7fffffffe2e8: 0x0000000000000000      0x00007fffffffe5a5  # argv[4](NULL), 이후 쓰레기값
```

### 2) 각 주소가 가리키는 실제 문자열 확인
```gdb
(gdb) x/s argv[0]
0x7fffffffe56b: "/home/linuxadmin/바탕화면/c_workspace/arguments"

(gdb) x/s argv[1]
0x7fffffffe59f: "1"

(gdb) x/s argv[2]
0x7fffffffe5a1: "2"

(gdb) x/s argv[3]
0x7fffffffe5a3: "3"

# NULL 포인터(argv[4]) 접근 시 에러 발생
(gdb) x/s argv[4]
0x0:    <error: Cannot access memory at address 0x0>
```
