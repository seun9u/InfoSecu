## 실습.[LAB] 구조체의 Call by Value, Call by Reference 사용하기

### 1. Call by Value (값에 의한 호출)
* **정의:** 함수 호출 시 구조체 변수 자체를 인수로 넘기는 방법
* **장점:** 원본 데이터를 보호할 수 있다. (출력 전용으로 안전함)
* **단점:** 구조체 크기가 클 경우 복사 과정에서 시스템 부하가 발생할 수 있다.
* **형식:**
  ```c
  struct member m;
  print_member(m);

  void print_member(struct member m) { ... }
  ```

### 2. Call by Reference (참조에 의한 호출)
* **정의:** 함수 호출 시 구조체 변수의 포인터(주소)를 인수로 넘기는 방법
* **장점:** - 원본 데이터를 직접 수정할 수 있다.
    - 데이터 크기가 커도 주소값(4~8byte)만 전달하므로 효율적이다.
* **형식:**
  ```c
  struct member m;
  print_member(&m);

  void print_member(struct member *m) { ... }
  ```

---

### 3. 실습 코드 (`struct_call_by_reference.c`)

```c
#include <stdio.h>

typedef struct
{
    int a;
    int b;
} POINT;

void swap(POINT *p2);

int main()
{
    POINT p1;
    p1.a = 1;
    p1.b = 2;
    /*
        p1 메모리 구조
        +----------------------+
        |   +-----+    +-----+ |
        | a |  1  |  b |  2  | |
        |   +-----+    +-----+ |   
        +----------------------+
        100 (주소 가점)
     */
    printf("main() Before: p1.a = %d, p1.b = %d\n", p1.a, p1.b); // 1 2
    swap(&p1);
    printf("main() After: p1.a = %d, p1.b = %d\n", p1.a, p1.b);  // 2 1

    return 0;
}

void swap(POINT *p2)
{
    /*
        p2 (포인터 변수)
        +-----+
        | 100 | (p1의 주소 저장)
        +-----+
     */
    int temp;
    temp  = p2->a;  // temp = 1;
    p2->a = p2->b;  // p2->a = 2;
    p2->b = temp;   // p2->b = 1;
    printf("swap(): p2->a = %d, p2->b = %d\n", p2->a, p2->b);  // 2 1
}
```

**실행 결과:**
```bash
$gcc -Wall -o struct_call_by_reference struct_call_by_reference.c$ ./struct_call_by_reference                 
main() Before: p1.a = 1, p1.b = 2
swap(): p2->a = 2, p2->b = 1
main() After: p1.a = 2, p1.b = 1
```
