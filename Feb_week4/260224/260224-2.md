# 🏷️ typedef문 분석

`typedef`는 기존 자료형(구조체/포인터 타입 등)에 새 이름(별칭)을 붙여서 더 짧고 읽기 쉽게 만드는 키워드이다. 새로운 자료형을 만드는 것이 아니라 별칭을 붙여서 또 다른 이름을 사용하는 것이다.

## 1. typedef 형식 및 예시

* **형식:** `typedef 원본 별명;`
* **예시:**
    * `typedef int INT;` ➔ `INT number;`
    * `typedef unsigned int size_t;` ➔ `size_t size;`

## 2. typedef 문을 사용해서 구조체를 정의하는 방법

### 0) 원래 형태 (typedef 미사용)
```c
struct jumsu {
    int kor;
    int eng;
};
struct jumsu j;
```

### 방법 1: 구조체 정의 후 typedef문으로 재정의
```c
struct jumsu {
    int kor;
    int eng;
};

typedef struct jumsu JUMSU;
JUMSU j2;
```

### 방법 2: 구조체 정의 시 typedef문으로 재정의
```c
typedef struct jumsu
{
    int kor;
    int eng;
} JUMSU;
```

### 방법 3: 구조체 정의 시 typedef문으로 재정의 (익명 구조체)
```c
typedef struct 
{
    int kor;
    int eng;
} JUMSU;

JUMSU jumsu;
jumsu.kor = 1;
jumsu.eng = 2;
```

## 3. 실습 1: 기본 자료형 별칭 사용 (`typedef1.c`)

### 소스 코드
```c
// typedef1.c
#include <stdio.h>

typedef int INT;
typedef char CHAR;

int main()
{
    INT a, b;
    CHAR c[] = "Hello C!";

    a = 1, b = 2;

    printf("size : a=%ld, b=%ld, c=%ld\n", sizeof(a), sizeof(b), sizeof(c));
    printf("value: a=%d, b=%d, c=%s\n", a, b, c);

    return 0;
}
```

### 실행 결과
```bash
$gcc -Wall -g -o typedef1 typedef1.c$ ./typedef1
size : a=4, b=4, c=9
value: a=1, b=2, c=Hello C!
```

## 4. 실습 2: 구조체 별칭 사용 (`struct6.c`)

### 소스 코드
```c
// struct6.c
#include <stdio.h>

typedef struct /* member */
{
    char name[20];
    int age;
} MEMBER;

int main()
{
    /* struct member */ MEMBER m = { "Hgd", 20 };
    printf("name = %s, age = %d\n", m.name, m.age);

    return 0;
}
```

### 실행 결과
```bash
$gcc -Wall -g -o struct6 struct6.c$ ./struct6
name = Hgd, age = 20
```
