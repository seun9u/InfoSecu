# 🏗️ C언어 구조체(Structure) 완벽 정리

구조체는 서로 다른 자료형을 하나로 묶어 사용자가 새롭게 정의하는 **사용자 정의 자료형**입니다.

## 1. 구조체의 특징
* **클래스의 원조:** C++, Java 등의 객체지향 언어에서 사용하는 '클래스'의 모태입니다.
* **이질적 데이터의 모임:** 배열이 '같은 자료형'의 모임이라면, 구조체는 **'서로 다른 자료형'**을 묶을 수 있습니다.
* **멤버(Member):** 구조체 내부에 선언된 변수들을 멤버라고 부릅니다.
* **제한 사항:** 구조체 내부에는 변수만 선언 가능하며, 함수는 직접 정의할 수 없습니다. (단, **함수 포인터**를 멤버로 사용하여 함수처럼 쓸 수 있습니다.)
* **틀(Template):** 구조체 정의 자체로는 메모리가 할당되지 않으며, 변수를 선언(인스턴스 생성)해야 비로소 메모리에 자리가 잡힙니다.

---

## 2. 구조체 정의 및 사용법

### 1) 구조체 정의 및 변수 선언
```c
struct 구조체명 {
    자료형 멤버1;
    자료형 멤버2;
};

// 변수 선언
struct 구조체명 변수명;
```

### 2) 멤버 접근 방법
* **일반 변수:** `.` (Dot) 연산자 사용 ➔ `a.x = 10;`
* **포인터 변수:** `->` (Arrow) 연산자 사용 ➔ `p->x = 10;`

## 3. 실습 1: 좌표 구조체 분석 (`struct1.c`)

### 소스 코드
```c
#include <stdio.h>

// 1. 구조체 정의 (일반적으로 main 밖에서 정의하여 전역적으로 사용)
struct point
{
    int x;
    int y;
};

int main()
{
    // 2. 구조체 변수 선언 (메모리 할당)
    struct point a;

    // 3. 구조체 멤버 사용
    a.x = 1;
    a.y = 2;

    // 4. 변수 출력
    printf("a.x = %d\n", a.x);
    printf("a.y = %d\n", a.y);

    return 0;
}
```

### 🐞 GDB 정밀 분석
GDB를 통해 구조체 변수 `a`가 메모리에 할당되고 값이 채워지는 과정을 확인합니다.



```gdb
(gdb) b main
(gdb) r
(gdb) disp a   # 구조체 변수 a를 자동 출력 설정
1: a = {x = -7432, y = 32767}   # 초기화 전에는 쓰레기 값이 들어있음

(gdb) n        # a.x = 1 실행 후
1: a = {x = 1, y = 32767}

(gdb) n        # a.y = 2 실행 후
1: a = {x = 1, y = 2}           # 멤버 변수들이 정상적으로 채워짐 확인
```

## 4. 실습 2: 회원 정보 관리 (`struct2.c`)

문자열 배열과 정수형을 묶어 하나의 데이터 단위(회원)를 관리하는 예제입니다.

### 소스 코드
```c
#include <stdio.h>

struct member
{
    char name[30];  // 이름
    char phone[30]; // 전화번호
    int  age;       // 나이
};

int main()
{
    struct member m;

    // 입력 단계
    printf("이름: ");
    scanf("%s", m.name);  // m.name은 배열의 이름이므로 주소(&) 생략 가능
    printf("전화번호: ");
    scanf("%s", m.phone);
    printf("나이: ");
    scanf("%d", &m.age);  // 일반 변수 멤버는 &를 붙여야 함

    // 출력 단계
    printf("이름: %s\n", m.name);
    printf("전화번호: %s\n", m.phone);
    printf("나이: %d\n", m.age);

    return 0;
}
```
