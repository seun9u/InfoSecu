# 🛠️ GDB TUI (Text User Interface) 및 어셈블리 설정 가이드

GDB TUI 모드는 터미널 화면을 분할하여 **소스 코드, 어셈블리 코드, 레지스터 상태**를 실시간으로 보면서 디버깅할 수 있게 해주는 강력한 기능입니다.

---

## 1. GDB TUI 실행 및 화면 관리

### 1) 실행 옵션
* **`gdb -tui <program>`**: GDB를 처음부터 TUI 모드로 시작합니다.
* **`gdb -q -tui <program>`**: 시작 시 나타나는 배너와 버전 메시지를 생략(`-q`, quiet)하고 깔끔하게 TUI 모드로 진입합니다.
    * 예: `$ gdb -q -tui ./for1`



### 2) 화면이 깨질 때 (Refresh)
디버깅 중 `printf` 출력이나 기타 이유로 화면 레이아웃이 깨지는 경우 화면을 새로고침해야 합니다.
* **명령어:** `refresh`
* **단축키:** **`Ctrl + L`** (가장 많이 사용됨)

### 3) TUI On/Off (GDB 실행 중)
GDB가 이미 실행 중인 상태에서도 TUI 모드를 켜거나 끌 수 있습니다.
* **`tui enable`**: TUI 모드 활성화
* **`tui disable`**: TUI 모드 비활성화 (일반 명령줄 모드로 복귀)
* **`help tui`**: TUI 관련 도움말 출력

---

## 2. 레이아웃(Layout) 설정
디버깅 목적에 따라 화면 구성을 자유롭게 바꿀 수 있습니다.

* **`layout src` (la sr)**: **소스 코드** 중심으로 화면 구성
* **`layout asm` (la as)**: **어셈블리 코드** 중심으로 화면 구성
* **`layout regs` (la re)**: 소스/어셈블리 창 위에 **레지스터(Register)** 창 추가
* **`layout split`**: **소스 코드와 어셈블리 코드**를 동시에 상하로 분할하여 표시
* **`help layout`**: 레이아웃 관련 도움말 출력

---

## 3. 윈도우 포커스(Focus) 이동
TUI 모드에서는 화살표 키가 기본적으로 소스 창의 스크롤 역할을 합니다. 명령어를 입력하려면 명령창(`cmd`)으로 포커스를 옮겨야 할 때가 있습니다.

* **`focus <창 이름>`**: 특정 창으로 포커스 이동
    * `focus src` / `focus asm` / `focus regs` / `focus cmd`
* **`focus next`**: 다음 창으로 포커스 이동
* **`focus prev`**: 이전 창으로 포커스 이동
* **`help focus`**: 포커스 관련 도움말 출력

---

## 4. 어셈블리어 문법 변환 (Disassembly Flavor)
GDB는 기본적으로 AT&T 문법을 사용하지만, 가독성이 좋은 Intel 문법으로 변경하여 확인할 수 있습니다.



### 1) 문법 확인 및 설정
* **`show disassembly-flavor`**: 현재 설정된 어셈블리 문법(att 또는 intel) 확인
* **`set disassembly-flavor intel`**: 문법을 **Intel** 스타일로 변경 (Windows/NASM 방식)
* **`set disassembly-flavor att`**: 문법을 **AT&T** 스타일로 변경 (Linux 기본 방식)

### 💡 Tip: AT&T vs Intel 주요 차이
| 특징 | AT&T (기본) | Intel (권장) |
| :--- | :--- | :--- |
| **데이터 방향** | `source, destination` | `destination, source` |
| **접두사** | 레지스터에 `%`, 상수에 `$` 사용 | 접두사 없음 |
| **값 크기 지정** | 접미사 사용 (mov**l**, mov**q**) | 키워드 사용 (DWORD PTR, QWORD PTR) |
