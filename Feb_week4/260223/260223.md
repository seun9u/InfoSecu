# 🛠️ GDB TUI (Text User Interface) 및 어셈블리 설정 가이드

GDB TUI 모드는 터미널 화면을 분할하여 **소스 코드, 어셈블리 코드, 레지스터 상태**를 실시간으로 보면서 디버깅할 수 있게 해주는 강력한 기능입니다.

---

## 1. GDB TUI 실행 및 화면 관리

### 1) 실행 옵션
* **`gdb -tui <program>`**: GDB를 처음부터 TUI 모드로 시작합니다.
* **`gdb -q -tui <program>`**: 시작 시 나타나는 배너와 버전 메시지를 생략(`-q`, quiet)하고 깔끔하게 TUI 모드로 진입합니다.
    * 예: `$ gdb -q -tui ./for1`



### 2) 화면이 깨질 때 (Refresh)
디버깅 중 `printf` 출력이나 기타 이유로 화면 레이아웃이 깨지는 경우 화면을 새로고침해야 합니다.
* **명령어:** `refresh`
* **단축키:** **`Ctrl + L`** (가장 많이 사용됨)

### 3) TUI On/Off (GDB 실행 중)
GDB가 이미 실행 중인 상태에서도 TUI 모드를 켜거나 끌 수 있습니다.
* **`tui enable`**: TUI 모드 활성화
* **`tui disable`**: TUI 모드 비활성화 (일반 명령줄 모드로 복귀)
* **`help tui`**: TUI 관련 도움말 출력

---

## 2. 레이아웃(Layout) 설정
디버깅 목적에 따라 화면 구성을 자유롭게 바꿀 수 있습니다.

* **`layout src` (la sr)**: **소스 코드** 중심으로 화면 구성
* **`layout asm` (la as)**: **어셈블리 코드** 중심으로 화면 구성
* **`layout regs` (la re)**: 소스/어셈블리 창 위에 **레지스터(Register)** 창 추가
* **`layout split`**: **소스 코드와 어셈블리 코드**를 동시에 상하로 분할하여 표시
* **`help layout`**: 레이아웃 관련 도움말 출력

---

## 3. 윈도우 포커스(Focus) 이동
TUI 모드에서는 화살표 키가 기본적으로 소스 창의 스크롤 역할을 합니다. 명령어를 입력하려면 명령창(`cmd`)으로 포커스를 옮겨야 할 때가 있습니다.

* **`focus <창 이름>`**: 특정 창으로 포커스 이동
    * `focus src` / `focus asm` / `focus regs` / `focus cmd`
* **`focus next`**: 다음 창으로 포커스 이동
* **`focus prev`**: 이전 창으로 포커스 이동
* **`help focus`**: 포커스 관련 도움말 출력

---

## 4. 어셈블리어 문법 변환 (Disassembly Flavor)
GDB는 기본적으로 AT&T 문법을 사용하지만, 가독성이 좋은 Intel 문법으로 변경하여 확인할 수 있습니다.



### 1) 문법 확인 및 설정
* **`show disassembly-flavor`**: 현재 설정된 어셈블리 문법(att 또는 intel) 확인
* **`set disassembly-flavor intel`**: 문법을 **Intel** 스타일로 변경 (Windows/NASM 방식)
* **`set disassembly-flavor att`**: 문법을 **AT&T** 스타일로 변경 (Linux 기본 방식)

### 💡 Tip: AT&T vs Intel 주요 차이
| 특징 | AT&T (기본) | Intel (권장) |
| :--- | :--- | :--- |
| **데이터 방향** | `source, destination` | `destination, source` |
| **접두사** | 레지스터에 `%`, 상수에 `$` 사용 | 접두사 없음 |
| **값 크기 지정** | 접미사 사용 (mov**l**, mov**q**) | 키워드 사용 (DWORD PTR, QWORD PTR) |

## 🛠️ 실습: GDB TUI 모드를 이용한 반복문 및 함수 분석

GDB TUI 모드를 활용하면 소스 코드의 실행 위치와 어셈블리 명령어를 실시간으로 대조하며 디버깅할 수 있습니다.

---

### 1. [LAB] for문 분석 (`for1.c`)

#### 소스 코드 및 실행
```c
// for1.c
#include <stdio.h>
int main(){
   int a = 1;
   for(a; a <= 5; a++)
   {
        printf("a = %d\n", a);
   }
    return 0;
}
```

#### GDB TUI 분석 과정
```bash
$gcc -g -o for1 for1.c$ gdb -q -tui ./for1
```
* **`(gdb) b main`**: 메인 함수에 중단점 설정
* **`(gdb) r`**: 프로그램 실행
* **`(gdb) n`**: 다음 줄 실행 (Next), 소스 창의 하이라이트가 이동하는 것을 확인
* **`Enter`**: 직전 명령어(`n`) 반복 실행
* **`(gdb) c`**: 남은 루프 끝까지 실행 (Continue)
* **`(gdb) q`**: GDB 종료

---

### 2. [LAB] while문 및 어셈블리 문법 변환 (`while1.c`)

TUI 모드에서 AT&T 문법과 Intel 문법의 차이를 직접 확인합니다.

#### 소스 코드
```c
// while1.c
#include <stdio.h>
int main(){
    int i = 1;              // 초깃값
    while(i <= 5)           // 조건식
    {
        printf("%d\n", i);  // 실행문
        i++;                // 증감식
    }
    return 0;
}
```

#### GDB TUI 상세 분석 로그
```bash
$gcc -g -o while1 while1.c$ gdb -q ./while1
```
* **`(gdb) tui enable`**: TUI 모드 켜기
* **`(gdb) layout split`**: 소스 코드와 어셈블리 창을 동시에 표시
* **`(gdb) focus cmd`**: 명령창으로 포커스를 옮겨 화살표 키로 명령어 히스토리 사용 가능하게 설정
* **`(gdb) b main` / `r`**: 실행
* **`(gdb) show disassembly-flavor`**: 현재 문법(기본 att) 확인
* **`(gdb) set disassembly-flavor intel`**: 가독성이 좋은 **Intel 문법**으로 변경
* **`(gdb) n`**: 소스와 어셈블리가 같이 한 줄씩 넘어가는 것을 관찰
* **`(gdb) set disassembly-flavor att`**: 다시 **AT&T 문법**으로 변경

### 3. [LAB] mystrlen 함수 정밀 분석 (`mystrlen.c`)

함수 내부에서 포인터 `s`의 주소 이동과 `count`의 변화를 TUI 모드에서 추적합니다.

#### GDB TUI 분석 로그
```bash
$ gdb -q ./mystrlen
```
* **`(gdb) tui enable`**
* **`(gdb) layout split`**
* **`(gdb) focus cmd`**
* **`(gdb) b main` / `r`**
* **`(gdb) n`**: `mystrlen(buf)` 호출 전까지 진행
* **`(gdb) p &buf[0]` / `p/x buf`**: 배열의 시작 주소 확인
* **`(gdb) s`**: `mystrlen` 함수 안으로 진입 (Step Into)
* **`(gdb) display s`**: 포인터 `s`의 주소 변화 감시 설정
* **`(gdb) display *s`**: 현재 포인터가 가리키는 문자 값 감시 설정
* **`(gdb) display count`**: 글자 수 카운트 감시 설정



#### 루프 및 리턴 확인
* **`n` / `Enter`**: 루프를 돌며 `count`가 올라가고 `s` 주소가 1씩 증가하는 것을 확인
* **`(gdb) x/xb s`**: 주소 `s` 위치에 NUL 문자(`0x00`)가 있는지 직접 확인
* **`(gdb) n`**: `return count;` 실행 후
* **`(gdb) i r eax`**: 리턴값이 담긴 레지스터(`eax`) 확인 (글자 수 출력)
* **`(gdb) c`**: 종료까지 진행
