# 🔗 이중 포인터와 포인터 배열의 함수 전달 분석

## 1. 이중 포인터의 기초 (`ex_pointer3.c`)

### 소스 코드
```c
// ex_pointer3.c
#include <stdio.h>
int main(){
    int i = 10;
    int *p1;
    int **p2;

    p1 = &i;
    p2 = &p1;

    printf("i    = %d\n", i);
    printf("*p1  = %d\n", *p1);
    printf("**p2 = %d\n", **p2);

    return 0;
}
```

## 2. 함수로 문자열 배열 전달하기 (`ex_pointer4.c`)

### 소스 코드
```c
// ex_pointer4.c
#include <stdio.h>

void print_pointer(char *p);
void print_pointer2(char **p);  // *p[] 와 동일

int main(){
    char ch[] = "Hello C!";
    char *ch2[] = { "Linux", "Windows", "Unix", "Mac", NULL };

    /*
       ch2 메모리 구조 (개념)
       [100|200|...|NULL]
         |   |
         |   +-- 200 "Windows"
         +-- 100 "Linux"
    */

    print_pointer(ch);
    print_pointer2(ch2);

    return 0;
}

void print_pointer(char *p) {
    printf("%s\n", p);
}

void print_pointer2(char **p) {
    for(int i=0; p[i] != NULL; i++)
    {
        printf("%s\n", p[i]);
        printf("%s\n", *(p+i)); // *(p+0) 포인터 연산 방식
    }
}
```

## 3. 시스템 아키텍처별 주소 체계 정리

* **32bit 시스템:**
    * 메모리 주소 크기: **4byte**
    * GDB 확인: `x/4xw address` (주소가 4, 8, c, 0 단위로 증가)
* **64bit 시스템:**
    * 메모리 주소 크기: **8byte**
    * GDB 확인: `x/2xg address` (주소가 0, 8 단위로 증가)
* **공통 사항:** * 자료형(`char`, `short`, `int` 등)에 관계없이 해당 자료형의 주소를 담는 **포인터 변수의 크기는 모두 동일**합니다 (64bit 환경에서는 모두 8byte).
