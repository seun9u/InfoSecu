# ☕ 자바(Java) 기초: 메서드와 데이터 타입 분석

자바에서 사용하는 **메서드(Method)**는 C언어의 **함수(Function)**와 동일한 개념입니다. 객체지향 프로그래밍으로 넘어오면서 명칭과 구조가 확장되었습니다.

---

## 1. C vs Java 프로그램 구조 비교

| 구분 | C언어 (절차지향) | Java (객체지향) |
| :--- | :--- | :--- |
| **최소 단위** | 함수 (Function) | 클래스 (Class) |
| **데이터 묶음** | 구조체 (struct) | 클래스 (Class) |
| **기능 단위** | 함수 (Function) | 메서드 (Method) |
| **특징** | 구조체 내부에 함수 정의 불가 | 클래스 내부에 데이터와 메서드 포함 가능 |

---

## 2. 자바 메서드(Method)의 종류



* **static 메서드:**
    - `new` 키워드로 객체를 생성하지 않아도 클래스 이름을 통해 바로 호출 가능합니다.
* **비 static 메서드 (인스턴스 메서드):**
    - 반드시 `new` 키워드를 사용하여 객체(인스턴스)를 생성한 후에만 호출할 수 있습니다.

---

## 3. 참(True)과 거짓(False)의 처리

| 언어 | 거짓(False) | 참(True) | 특징 |
| :--- | :--- | :--- | :--- |
| **C** | 0 | 1 (또는 0 외 나머지) | `true`, `false` 키워드 제공 안 함 |
| **Java / C++** | 0 / `false` | 1 / `true` | 전용 타입(`boolean`)과 키워드 존재 |
| **Python** | `False` | `True` | 첫 글자가 대문자인 키워드 사용 |

---

## 4. 데이터 타입의 차이 (C vs Java)

### 1) 문자열 저장
* **C:** `char *message = "Hello";` (포인터 사용)
* **Java:** `String message = "Hello";` (`String` 클래스 사용)

### 2) 문자(char) 처리
* **C:** `char ch = '가';` ➔ **에러 발생** (한글은 1바이트에 담을 수 없음)
* **Java:** `char ch = '가';` ➔ **가능** (자바의 char는 유니코드 기반 2바이트)

### 3) 논리형 (boolean)
* `true`(참)와 `false`(거짓) 두 가지만을 저장하는 타입입니다.

## 5. 실습 예제 분석 (`Ex02_19.java`)

### 소스 코드
```java
package section01;

public class Ex02_19 {

    public static void main(String[] args) {
        // boolean 타입 선언 및 초기화
        boolean isStudent = true;
        
        // ! 연산자 (NOT): 논리값을 반대로 뒤집음
        if(!isStudent) // isStudent가 false이면 참
            System.out.println("저는 학생입니다.");
        else           // isStudent가 true이면 실행
            System.out.println("저는 학생이 아닙니다.");
    }
}
```

### 📍 분석 결과
* **`isStudent`**가 `true`이므로, **`!isStudent`**는 `false`가 됩니다.
* 따라서 `else` 블록이 실행되어 **"저는 학생이 아닙니다."**가 출력됩니다.

## 6. 문자열 변수 선언 예시
```java
String a = "Apple";
```
* 자바에서 `String`은 기본 자료형이 아니라 **참조 자료형(클래스)**입니다. 대문자로 시작한다는 점에 유의 !!
