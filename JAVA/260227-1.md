# 🏗️ 자바 핵심 실습: 클래스 구조, 생성자, 멤버 및 정보 은닉

## 1. 클래스 기본 구조와 생성자 (Constructor)
클래스는 객체의 설계도이며, 생성자는 객체가 생성될 때 호출되어 초기화 작업을 수행합니다.

* **기본 생성자 (Default Constructor)**: 클래스에 생성자가 없으면 자바 컴파일러가 아무 일도 하지 않는 빈 생성자를 자동으로 추가합니다.
* **초기화 방식 비교**:
    * **생성자 미활용**: 객체 생성 후 멤버 변수(`c.wheel = 500`)에 수동으로 값을 대입합니다.
    * **생성자 활용**: `new Cat(500)`과 같이 객체 생성과 동시에 초기값을 전달하여 효율적으로 설정합니다.


### [실습 코드] Cat.java & Ex07_03.java
```java
// Cat.java
package package07;

public class Cat {
    int wheel = 4;
    
    void ride() {
        System.out.println("달립니다.");
        System.out.println("씽씽씽");
    }
}

// Ex07_03.java
package package07;

public class Ex07_03 {
    public static void main(String[] args) {
        // 객체 생성
        Cat c = new Cat();
        c.ride();  // 메서드 호출
        c.ride();
        c.ride();
        System.out.println(c.wheel);  // 필드 출력
    }
}
```

## ⚙️ 2. 정적 멤버(static) vs 인스턴스 멤버
`static` 키워드 유무에 따라 필드와 메서드가 메모리에 할당되는 시점과 사용법이 달라집니다.


| 구분 | 정적 멤버 (static) | 인스턴스 멤버 (non-static) |
| :--- | :--- | :--- |
| **소속** | 클래스 자체에 소속 | 생성된 객체(인스턴스)에 소속 |
| **호출 방법** | `클래스명.멤버` (객체 생성 불필요) | `객체명.멤버` (new 객체 생성 필수) |
| **특징** | 모든 객체가 데이터를 공유함 | 객체마다 독립적인 값을 가짐 |

### [실습 코드] Student.java & Ex07_15.java
```java
// Student.java
package package07;

public class Student {
    static String schoolName = "코리아 대학원";  // 정적 필드
    String studentName;                         // 인스턴스 필드
    
    static void goToSchool() {                  // 정적 메서드
        System.out.println("학교에 갑니다.");
    }
    
    void hello() {                              // 인스턴스 메서드 
        System.out.println("이름: " + studentName);
    }   
}

// Ex07_15.java
package package07;

public class EX07_15 {
    public static void main(String[] args) {
        // 인스턴스 멤버 사용을 위한 객체 생성
        Student stu1 = new Student();
        stu1.studentName = "김고이";
        stu1.hello();

        // 정적 멤버 사용 (클래스 이름으로 직접 호출)
        System.out.println("학교: " + Student.schoolName);
        Student.goToSchool();
    }
}
```

## 🔒 3. 정보 은닉과 Getter/Setter 메서드
중요한 필드를 `private`으로 숨기고, 메서드를 통해서만 접근하게 하여 데이터의 무결성을 보장합니다.

* **Setter**: 값을 검증하고 설정하는 문지기 역할 (예: 나이에 음수 입력 방지).
* **Getter**: 숨겨진 데이터를 안전하게 읽어오는 역할.

### [실습 코드] Person.java & Ex08_21.java
```java
// Person.java
package section08;

public class Person {
    private int age;  // 외부 접근 금지
    
    // Setter: 메서드를 통해 값을 필터링하여 설정
    void setAge(int num) {
        if(num < 0) {
            System.out.println("1이상 입력하세요.");
            return;
        } else {
            age = num;
        }
    }
    
    // Getter: 안전하게 값을 반환
    int getAge() {
        return age;
    }
}

// Ex08_21.java
package section08;

public class Ex08_21 {
    public static void main(String[] args) {
       Person kim = new Person();
       
       // kim.age = -30; // 접근 제어로 인해 에러 발생
       kim.setAge(30);  // Setter를 이용한 정상 설정
       System.out.println("kim의 나이: " + kim.getAge());
    }
}
```
