# 🏗️ 객체지향 프로그래밍과 클래스 구조 분석

자전거, 자동차와 같은 복잡한 시스템을 구현하기 위해, 기존의 절차지향 프로그래밍(C)의 부족한 점을 보완하며 발전한 것이 객체지향 프로그래밍(Java)입니다.

### 1. 프로그래밍 패러다임의 변화
* **절차지향 (C):** 순서에 따른 실행을 중시하며 클래스 이전에는 구조체 형태를 사용했습니다.
* **객체지향 (Java):** 실제 운영 환경에서 절차지향보다 효율적임을 확인하며 발전했습니다. 구조체에 데이터와 함수를 묶어 클래스로 진화시켰습니다.



---

### 2. 라이브러리(Library)의 비교
* **C언어:** 함수를 하나로 모아놓은 것
* **JAVA:** 클래스(Class)를 하나로 모아놓은 것

---

### 3. 객체를 생성하고 사용하는 순서


1. **클래스 선언(정의):** `접근제한자 class 클래스명 { ... }`
2. **클래스명 생성:** `클래스명 변수명;` (참조 변수 선언)
3. **객체 (인스턴스) 생성:** `변수명 = new 클래스명();` (다른 클래스에서 생성)
4. **필드 사용:** `객체명.필드명;`
5. **메서드 사용:** `객체명.메서드();`
