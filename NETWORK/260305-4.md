# 🌐 L3 스위치 라우팅 실습: 서로 다른 대역 간 통신 설정

L3 스위치에서 기본 L2 기능을 끄고, 각 포트를 라우터의 인터페이스처럼 사용하여 두 네트워크(192.168.1.0/24 및 192.168.2.0/24)를 연결하는 실습입니다.

---

## 1. 초기 확인 및 라우팅 기능 활성화

L2 스위치 상태에서는 라우팅 테이블이 보이지 않거나 동작하지 않습니다.

```ios
Switch> enable
Switch# configure terminal

// 1. 라우팅 기능 켜기 (L2 스위치 -> L3 스위치 전환)
Switch(config)# ip routing

// 2. 라우팅 테이블 활성화 확인
Switch(config)# do show ip route
// (Codes 목록과 함께 'Gateway of last resort is not set' 메시지가 나오면 정상)
```

---

## 2. 인터페이스별 IP 주소 설정 (L3 Routed Port)

포트를 `no switchport` 명령어로 전환하여 라우터 포트처럼 동작하게 만든 후 IP를 할당합니다.



### 📍 FastEthernet 0/1 설정 (1번 대역 게이트웨이)
```ios
Switch(config)# interface fa0/1
Switch(config-if)# no switchport                          // L2 포트를 L3 포트로 변경
Switch(config-if)# ip address 192.168.1.254 255.255.255.0  // IP 주소 설정
Switch(config-if)# no shutdown                             // 인터페이스 활성화
```

### 📍 FastEthernet 0/2 설정 (2번 대역 게이트웨이)
```ios
Switch(config)# interface fa0/2
Switch(config-if)# no switchport                          // L2 포트를 L3 포트로 변경
Switch(config-if)# ip address 192.168.2.254 255.255.255.0  // IP 주소 설정
Switch(config-if)# no shutdown                             // 인터페이스 활성화
Switch(config-if)# end
```

## 3. 설정 저장 및 라우팅 테이블 최종 확인

```ios
// 설정값 저장 (재부팅 시 초기화 방지)
Switch# write memory

// 라우팅 테이블 확인 (Connected 네트워크 등록 확인)
Switch# show ip route

// 출력 결과 예시:
// C    192.168.1.0/24 is directly connected, FastEthernet0/1
// C    192.168.2.0/24 is directly connected, FastEthernet0/2
```

## 4. 통신 테스트 (Ping Test)

192.168.1.x 대역의 PC에서 192.168.2.1로 통신을 시도합니다.

```cmd
C:\> ping 192.168.2.1

Pinging 192.168.2.1 with 32 bytes of data:

Request timed out.  <-- 첫 번째 패킷은 ARP 과정에서 타임아웃이 발생할 수 있음
Reply from 192.168.2.1: bytes=32 time<1ms TTL=127
Reply from 192.168.2.1: bytes=32 time<1ms TTL=127
Reply from 192.168.2.1: bytes=32 time<1ms TTL=127
```
