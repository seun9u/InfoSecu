# 🌐 패킷 트레이서 시뮬레이션 및 이더넷(CSMA/CD) 상세 분석

## 1. Packet Tracer Simulation 모드 활용
시뮬레이션 모드는 네트워크 분석을 위해 필수적입니다. 프로그래밍 언어의 **디버깅**과 같아서 데이터 전송의 세부 과정을 분석할 수 있습니다.

* **Realtime 모드**: 네트워크 토폴로지에서 각 장비들의 실제 동작을 실행하는 모드입니다. 시각적으로 패킷의 분석이나 전송 과정을 확인할 수 없습니다.
* **Simulation 모드**: 각 장비의 동작 상황을 분석할 때 사용하는 모드입니다. 시각적으로 패킷 분석/전송을 확인할 수 있어 입문자에게 필수적입니다.

### 🛠️ 시뮬레이션 사용 방법
1. 오른쪽에 **[Simulation]** 버튼 클릭 > **[Edit Filters]** 클릭.
2. [IPv4], [IPv6], [Misc] 탭에서 원하는 프로토콜을 선택합니다.
3. **[Show All/None]**: 토글키로 한 번씩 클릭하면 프로토콜 전체 선택/삭제가 가능합니다.
4. **분석 예시**:
    - **DNS 및 WEB 서버 분석**: [전체해제] > [DNS], [HTTP], [TCP] 선택 후 창을 닫습니다. 클라이언트에서 웹서버 접속 시 동작을 확인할 수 있습니다.
    - **Ping 통신 분석**: [전체해제] > [ICMP] 선택 후 PC의 CLI(Command Prompt)에서 `ping 192.168.1.2`를 실행하여 동작을 확인합니다.

## 2. 허브(HUB)의 동작 특성과 한계
허브는 **1계층(물리 계층)** 장비로, 전기적인 신호가 들어오면 수신 포트를 제외한 **모든 포트로 전송**하는 특징이 있습니다.

* **동작 원리**: 들어온 포트를 제외한 모든 포트에 전기적인 신호가 전송됩니다.
* **속도 저하의 원인**: 과거 허브 사용 시 속도가 느렸던 이유는 낮은 PC 사양과 통신 속도 탓도 있지만, **전기적 신호가 양방향에서 동시에 들어올 때 발생하는 충돌(Collision)**이 가장 큰 원인이었습니다.
* **해결 노력**: 충돌 문제를 해결하기 위해 데이터를 전송하기 전 선로에 신호가 있는지 먼저 확인하는 **CSMA/CD 방식**이 도입되었고, 이후 이를 근본적으로 해결한 **Switch**가 등장하게 되었습니다.

## 3. CSMA/CD (Carrier Sense Multiple Access with Collision Detection)
유선 이더넷(IEEE 802.3) 환경에서 데이터 충돌을 감지하고 재전송하여 효율적인 통신을 보장하는 방식입니다.



### 📍 핵심 동작 원리
1. **Carrier Sense (반송파 감지)**: 데이터를 전송하기 전, 네트워크 매체(선로)의 신호를 감지하여 사용 중인지 확인합니다. 사용 중이면 대기하고, 비어있으면 전송을 시작합니다.
2. **Multi Access (다중 접근)**: 여러 노드가 하나의 통신 채널을 공유하여 사용합니다.
3. **Collision Detection (충돌 탐지)**: 데이터를 전송하면서 동시에 충돌 여부를 감지합니다. 충돌이 감지되면 즉시 전송을 중단하고 **'재밍(Jamming)'** 신호를 보내 충돌 발생을 알립니다. 이후 랜덤 시간(**Backoff**) 동안 대기했다가 재전송합니다.

## 4. MAC 주소 체계와 전송 방식
이더넷 헤더에서 목적지를 식별하는 방식은 크게 두 가지로 나뉩니다.

* **브로드캐스트(Broadcast) MAC 주소**: 네트워크의 모든 장비에게 전송합니다.
    - 목적지 MAC 주소: **`FFFF.FFFF.FFFF`**
* **유니캐스트(Unicast) MAC 주소**: 특정 목적지 하나에게만 전송합니다.
    - 목적지 MAC 주소 예시: `0002.17DC.1DB0`, `1234.5678.abcd`
