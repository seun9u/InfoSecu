# 🌐 L3 스위치 라우팅 설정 및 명령어 상세 분석

L3 스위치는 기본적으로 L2 스위치처럼 동작하지만, 설정을 통해 라우터와 같은 3계층 기능을 수행할 수 있습니다.

---

## 1. L3 스위치 라우팅 활성화 단계
L3 스위치에서 서로 다른 네트워크 간 통신(라우팅)을 하려면 다음 설정이 필수입니다.

1. **글로벌 설정 모드**: `ip routing` 명령으로 라우팅 기능 활성화.
2. **인터페이스 선택**: 라우팅을 수행할 포트로 이동.
3. **L3 포트 전환**: `no switchport` 명령 실행.
4. **IP 설정 및 활성화**: `ip address [주소] [마스크]` 설정 후 `no shutdown`.

---

## 2. switchport vs no switchport



### 📍 switchport (Default 설정)
인터페이스를 **L2 스위치 포트**로 동작시키는 모드입니다.
* **특징**: 포트에 직접 IP 주소를 설정할 수 없으며, 대신 VLAN 할당이나 트렁크 설정을 수행합니다.
* **주요 기능**: `access/trunk` 설정, `STP`, `Port-Security` 적용 등.
* **설정 예시**:
  ```ios
  interface fa0/1
  switchport mode access
  switchport access vlan 10
  ```
  *※ 이 상태에서 `ip address`를 설정하면 에러가 발생하며, IP는 `interface vlan 1` 같은 SVI에 설정해야 합니다.*

### 📍 no switchport
인터페이스를 **L3 라우티드 포트(Routed Port)**로 전환하는 명령어입니다.
* **특징**: 라우터의 인터페이스처럼 동작하며, 포트에 **직접 IP 주소 설정**이 가능합니다.
* **결과**: 설정된 주소는 라우팅 테이블에 `connected`(직접 연결된 네트워크)로 등록됩니다.
* **설정 예시**:
  ```ios
  interface fa0/1
  no switchport
  ip address 192.168.1.254 255.255.255.0
  no sh
  ```
  *※ 이 상태에서는 `switchport mode access/trunk` 같은 L2 명령어를 사용할 수 없습니다.*

---

## 3. ip routing vs no ip routing (전역 설정)



| 구분 | ip routing | no ip routing (Default) |
| :--- | :--- | :--- |
| **동작 모드** | **L3 라우터** 모드 ON | **L2 스위치** 모드 |
| **핵심 기능** | 수신된 패킷을 다른 목적지 네트워크로 전송(라우팅) | 라우팅을 수행하지 않고 스위칭만 수행 |
| **활용 사례** | 서로 다른 IP 대역(1.0/24 ↔ 2.0/24) 간 게이트웨이 역할 수행 시 | 단순 Access 스위치로 사용하며 라우팅은 상단 라우터/방화벽에 맡길 때 |
| **IP 용도** | 네트워크 간 데이터 전송(Gateway) | 관리 목적(SSH/Telnet/Web)으로만 사용 |

* **참고**: `no ip routing` 상태에서 원격 관리를 하려면 스위치에 `ip default-gateway` 설정이 별도로 필요합니다.
* **패킷트레이서 제약**: 시스코 패킷트레이서에서는 관리용 접속으로 `SSH`와 `Telnet`만 지원합니다.
