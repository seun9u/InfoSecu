# 🛠️ [파트 2] 실습.[LAB] 스위치/라우터 초기화 및 주요 기술 개념

## 1. 장비 기본 설정 및 초기화 방법
네트워크 장비를 처음 사용하거나 설정을 완전히 지우고 다시 시작할 때 사용하는 절차입니다.

### 📍 기본 환경 설정
```ios
! R1 라우터 기본 설정
Router> enable
Router# configure terminal
Router(config)# no ip domain-lookup   // 명령어 오타 시 DNS 서버 조회 방지
Router(config)# hostname R1           // 장비 이름 변경
Router(config)# exit
Router# write memory                  // 현재 설정을 NVRAM에 저장
```

### 📍 장비 공장 초기화 (Reset)
```ios
! 저장된 설정 파일 삭제 및 재부팅
R1# erase startup-config              // NVRAM에 저장된 설정 파일 삭제
R1# reload                            // 장비 재시작 (초기 상태로 복구)
```

## 2. 네트워크 핵심 기술 개념

### 📍 ARP (Address Resolution Protocol)
* **정의**: 상대방의 IP 주소는 알고 있지만, 실제 데이터를 전달하기 위한 **MAC 주소**를 모를 때 이를 알아오는 프로토콜입니다.
* **동작**: 상대방의 MAC 주소를 모르면 ARP가 자동으로 동작하여 정보를 수집합니다.

### 📍 IP 주소 체계의 기초
* **네트워크 주소**: IP 주소 범위 중에서 가장 처음에 시작하는 주소가 해당 네트워크의 주소가 됩니다.
* **2진수 변환 예시**: `00000000`은 0, `11111111`은 255를 의미합니다.

## 3. 실습 관련 기술 문답 (Q&A)



* **Q: ARP 프로토콜이 동작하는 이유는 무엇인가요?**
    * **A**: 통신을 위해 2계층 주소(MAC)가 필수적인데, 상대방의 MAC 주소를 모르는 상태에서는 패킷을 완성하여 보낼 수 없기 때문입니다.
* **Q: ARP 요청 시 Ethernet 헤더와 ARP 헤더의 목적지 주소는 무엇인가요?**
    * **A**: Ethernet 헤더의 목적지 MAC은 **`FFFF.FFFF.FFFF`**(브로드캐스트)가 들어가며, ARP 헤더의 Target Hardware Address는 **`0000.0000.0000`**으로 채워 보냅니다.
* **Q: 스위치의 5가지 핵심 기능은 무엇인가요?**
    * **A**: 1. Learning(학습), 2. Flooding(플러딩), 3. Forwarding(포워딩), 4. Filtering(필터링), 5. Aging(에이징)입니다.
* **Q: PC0에서 PC1로 핑을 보낼 때 ARP가 아닌 ICMP가 먼저 동작하는 이유는?**
    * **A**: 이미 이전에 통신한 기록이 있어 **ARP 캐시 테이블**에 상대방의 MAC 주소가 이미 저장되어 있기 때문입니다.

---

### 📍 라우팅 테이블의 중요성
* 라우터는 패킷이 들어오면 목적지 주소를 보고 자신의 라우팅 테이블을 확인합니다.
* 테이블에 목적지 네트워크가 있으면 해당 인터페이스로 보내고, 없으면 패킷을 **폐기**합니다.
* 라우팅 테이블이 정확하지 않으면 네트워크 전체에 문제가 생길 수 있으므로 완벽하게 관리되어야 합니다.
