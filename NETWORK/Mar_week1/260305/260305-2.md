# 🌐 네트워크 계층별 데이터 처리 및 장비 매커니즘 상세 분석

## 1. 데이터 수신 시 계층별 상세 동작 (PDU 분석)
장비가 데이터를 수신했을 때 각 계층(Layer)에서 일어나는 내부 프로세스입니다.

### [1계층] 물리 계층 (Physical Layer)
* **동작**: `FastEthernet0` 포트가 물리적 신호인 프레임을 수신합니다.

### [2계층] 데이터 링크 계층 (Data Link Layer)
* **MAC 주소 일치 시 (예: 목적지 192.168.1.2)**:
    * 프레임의 목적지 MAC 주소가 수신 포트의 MAC 주소, 브로드캐스트 주소, 또는 멀티캐스트 주소와 일치하는지 확인합니다.
    * 일치하면 장치는 이더넷 프레임에서 상위 계층 데이터인 **PDU를 디캡슐화(Decapsulates)** 합니다.
* **MAC 주소 불일치 시 (예: 목적지 192.168.1.3)**:
    * 목적지 MAC 주소가 수신 포트 정보와 일치하지 않으면, 장치는 해당 프레임을 **폐기(Drop)** 합니다.

### [3계층] 네트워크 계층 (Network Layer)
* **IP 주소 일치 시 (예: 목적지 192.168.1.2)**:
    * 패킷의 목적지 IP 주소가 장치의 IP 주소 또는 브로드캐스트 주소와 일치하면 패킷을 디캡슐화합니다.
    * 해당 패킷이 **ICMP 패킷**인 경우, ICMP 프로세스가 이를 처리하며 **에코 요청(Echo Request)** 메시지를 수신했음을 인식합니다.

## 2. 스위칭 테이블과 컬리전 도메인 원리

### 📍 스위칭 테이블 (MAC Address Table)
스위치는 메모리에 MAC 주소를 저장하는 공간인 **MAC Address Table**을 관리합니다.
* **초기 상태**: PC들이 연결되어 있어도 실제 통신(ping, 웹 접속 등)이 발생하기 전까지는 테이블에 아무런 값이 없습니다.
* **학습 및 등록**: 프레임이 스위치로 들어오면 디캡슐레이션을 통해 **출발지 주소**를 확인하고, 해당 주소를 프레임이 들어온 포트 번호와 함께 테이블에 등록합니다.
    * 예시: `aaaa.1234.5678 - fa0/1`
* **핵심**: 서로 간에 실제 통신이 일어나지 않으면 스위치의 MAC 주소 테이블에 정보가 입력될 수 없습니다.



### 📍 컬리전 도메인 (Collision Domain) 비교
* **허브 (Hub)**: 모든 포트가 전기적으로 연결되어 있어 전체가 **1개의 컬리전 도메인**을 형성합니다.
* **스위치 (Switch)**: 각 포트별로 독립적인 컬리전이 존재합니다. 4대의 PC가 연결된 스위치라면 **컬리전 도메인은 4개**가 됩니다.

## 3. 케이블 특성 및 L2/L3 장비 차이점

### 📍 UTP 케이블의 송수신 (TX / RX)
* **TX (Transmit)**: 장비가 전기 신호(데이터)를 외부로 내보내는 송신 선로 쌍입니다.
* **RX (Receive)**: 외부의 전기 신호를 받아들이는 수신 선로 쌍입니다.

### 📍 L2 스위치 vs L3 스위치 (다른 네트워크 통신)
* **통신 요건**: 다른 네트워크에 있는 노드와 통신하려면 IP 주소를 인식하고 라우팅을 수행하는 **L3 장비**가 필요합니다.
* **L2 스위치**: 라우팅 기능이 없어 다른 네트워크로 프레임을 전송할 수 없습니다.
* **L3 스위치**: 라우팅을 통해 다른 네트워크로 패킷을 전송할 수 있습니다.
* **Multilayer Switch**: L2와 L3 기능을 모두 수행할 수 있는 스위치입니다.
    * 참고: 패킷트레이서에서 L3 스위치를 연결하면 초기에는 L2 모드로 동작합니다.

### 📍 시스코 스위치 모델별 하드웨어 특징
* **3560 24PS**: FastEthernet(100M) 24포트와 GigabitEthernet(1G) 2포트를 제공합니다.
* **3650 24PS**: GigabitEthernet 24포트와 4포트를 제공하며, 사용 시 **전원 모듈**을 직접 장착해야 합니다.
