# 💻 프로그래밍 vs 네트워크 학습 가이드 및 네트워크 구조 분석

프로그래밍과 네트워크는 학습 방식과 목적에서 뚜렷한 차이를 보입니다.

## 1. 학습 방식 비교
* **프로그래밍**
    * 용어와 문법을 암기합니다.
    * 이미 존재하는 프로그램을 똑같이 만들어 보며 연습합니다.
    * 최종적으로는 세상에 **없는 것을 새롭게 창조**하는 것이 목적입니다.
* **네트워크**
    * 용어와 명령어를 암기합니다.
    * 가상 환경(Packet Tracer 등)에서 직접 구축해보고 분석합니다.
    * 이미 **구축되어 있는 표준 기술을 능숙하게 사용**하는 것이 목적입니다.

---

## 2. 네트워크 계층 구조 (Cisco 3-Layer 모델)
네트워크는 효율적인 관리와 확장을 위해 계층 구조로 설계됩니다.



| 계층 (Layer) | 주요 장비 | 식별 주소 및 특징 |
| :--- | :--- | :--- |
| **엑세스 (Access)** | **L2 스위치** | **MAC 주소** 기반 통신 (IP, Port 인식 불가) |
| **디스트리뷰션 (Distribution)** | **L3 스위치** | **MAC, IP 주소** 기반 통신 (Port 인식 불가) |
| **코어 (Core)** | **L3, Router** | **MAC, IP 주소** 기반의 고속 데이터 전송 |

---

## 3. MAC 주소 (물리적 주소)의 유일성
MAC 주소는 네트워크 장비의 고유 번호로, 비유하자면 **대학생들의 학번**과 같습니다.
* **충돌 주의**: 동일한 LAN 상에서 MAC 주소는 반드시 **유일**해야 하며, 중복될 경우 통신 장애가 발생합니다.

### 📍 벤더(제조사)별 MAC 주소 접두어 예시
MAC 주소의 앞쪽 6자리는 제조사를 식별하는 코드입니다.



* **VMware (가상머신)**
    * 주소 형식: `00:0C:29:??:??:??`
    * 제조사: VMware, Inc. (Palo Alto, CA, US)
* **삼성전자 (안드로이드 휴대전화)**
    * 주소 형식: `BC:93:07:??:??:??`
    * 제조사: Samsung Electronics Co.,Ltd (Gumi, KR)
* **Apple (아이폰)**
    * 주소 형식: `FC:31:5D:??:??:??`
    * 제조사: Apple, Inc. (Cupertino, CA, US)

---

### 🛠️ 장치별 확인 방법
1. **VMware**: 가상머신 `Settings` > `Network Adapter` > `Advanced...` 에서 확인 가능합니다.
2. **안드로이드**: `설정` > `휴대전화 정보` > `상태 정보` > `Wi-Fi MAC 주소`에서 확인 가능합니다.
3. **아이폰**: `설정` > `일반` > `정보` > `Wi-Fi 주소`에서 확인 가능합니다.
